# 网站架构与功能梳理（模块职责版）

> 目标：把“谁负责什么、链路怎么走、问题落在哪层”一次讲清楚

---

## 1. 总体分层

1. 展示层（HTML + CSS）
- 负责页面结构与视觉表达
- 文件：index / ai / wenku / nianfo / gongxiu / admin 多入口页面

2. 前端应用层（src/js）
- 负责状态、路由、播放器、页面逻辑、AI/文库交互
- 特点：Vanilla JS + ES Modules，按功能拆分模块

3. 边缘 API 层（functions）
- 负责统一 API 入口、参数校验、鉴权、业务编排
- 特点：Cloudflare Pages Functions

4. 数据与媒体层（D1 + R2 + Worker）
- D1：结构化业务数据
- R2：音频对象存储
- audio worker：音频域名与分发链路

5. 平台能力层（PWA + 缓存 + 部署）
- Service Worker、manifest、离线策略、Cloudflare 部署

---

## 2. 前端模块职责

## 2.1 核心基建

1. 状态中心
- 职责：保存当前播放序列、当前曲目、用户设置、主题/语言
- 价值：避免模块互相写全局变量

2. 路由与页面切换
- 职责：页面视图切换、参数处理、入口分流
- 价值：把导航逻辑与业务逻辑解耦

3. 通用能力
- 职责：请求封装、工具函数、埋点、DOM 访问封装

## 2.2 播放器域（当前 P0 重点）

1. 播放内核
- 职责：音频元素控制、播放/暂停/切歌、倍速、进度

2. 会话隔离
- 职责：防止快速切歌时异步回调串写
- 现状：已抽离基础会话管理模块并接入主流程

3. 缓冲与恢复
- 职责：弱网下缓冲提示、重试、错误态与恢复

4. 列表与 UI 联动
- 职责：当前项高亮、播放列表渲染、进度/状态展示

## 2.3 AI 域

1. 问答交互
- 职责：提问、流式/分段回复展示、错误兜底

2. 引用链路
- 职责：回答中的来源关联、跳转到原文或相关内容

## 2.4 阅读器域

1. 内容展示
- 职责：章节、正文、排版、目录导航

2. 阅读状态
- 职责：阅读进度、偏好设置、回到上次位置

---

## 3. 后端模块职责

1. API 路由入口
- 职责：统一请求入口，分发到对应业务模块

2. 业务模块（functions/lib）
- 职责：内容管理、消息、AI 路由、文库路由、转录路由

3. 分享路由（functions/share）
- 职责：外部分享链接与兼容展示

4. 音频 Worker
- 职责：音频资源访问与分发策略

---

## 4. 数据与缓存职责

1. D1 数据库
- 职责：系列、曲目、内容元数据、业务记录

2. R2 对象存储
- 职责：音频文件与静态媒体对象

3. Service Worker 缓存
- 职责：静态资源、数据请求、音频缓存分层控制

4. 前端音频缓存
- 职责：短音频本地缓存、离线回退

---

## 5. 关键功能链路（用户视角）

## 5.1 听

1. 用户选曲
2. 前端决定播放策略（弱网/缓存/直连）
3. 播放器启动会话并加载音源
4. 播放中进行缓冲监测与恢复
5. 记录进度并更新 UI

## 5.2 问

1. 用户在 AI 页发问
2. 前端请求 Functions AI 路由
3. 后端调用模型与知识来源
4. 前端渲染回答与来源
5. 用户可跳转到原文/继续追问

## 5.3 读

1. 用户进入文库/章节
2. 前端拉取内容并渲染阅读器
3. 用户调整阅读参数（字号/行高等）
4. 系统保存阅读进度与偏好

---

## 6. 当前播放器问题在架构中的落点

1. 弱网延迟
- 落点：播放器策略层 + 缓存层

2. 快速切歌串写
- 落点：播放器会话隔离层

3. 页面发僵
- 落点：播放器 UI 更新层（高频写入）

4. iOS 恢复异常
- 落点：播放器恢复策略层

---

## 7. 下一步落地顺序（只针对播放器）

1. 继续统一异步回调门禁（session.guard）
2. 抽离 telemetry（可观测性）
3. 抽离 UI 更新模块（降复杂度）
4. 进行弱网专项回归与指标化验收
