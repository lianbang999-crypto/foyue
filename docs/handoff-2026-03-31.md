## 8. 晚间追加更新（播放器与架构）

### 8.1 播放器弱网与卡顿感知参数已下调（体感优先）
在 `src/js/player.js` 已完成以下阈值调整：

1. 正常网络卡顿检测：12s -> 3.5s
2. 弱网卡顿检测：25s -> 6s
3. 慢加载提示：15s -> 3s
4. 首播硬超时：20s -> 9s
5. 恢复硬超时：9s
6. 重试兜底超时：12s

目标是把“用户体感卡住”的反馈前置，不再长时间无反馈等待。

### 8.2 播放会话隔离模块已独立
已存在独立会话管理文件：`src/js/player/session.js`。

当前会话模块职责：

1. begin：切歌时递增会话 ID
2. isCurrent：判定异步回调是否仍属于当前会话
3. guard：只允许当前会话执行异步回写

作用是降低“快速切歌时上一首异步回写下一首状态”的风险。
