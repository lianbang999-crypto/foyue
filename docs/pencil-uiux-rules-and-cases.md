# Pencil UI/UX 规则与案例参考（音频 + AI + 阅读器）

> 适用对象：净土法音（音频网站，含 AI 问法与阅读器）
> 用途：给 Pencil/设计智能体直接喂设计约束与参考方向

---

## 1. 设计目标

你的网站本质是三件事：

1. 听：稳定播放、快速恢复、弱网可用
2. 问：AI 基于原文引用回答，而不是自由发挥
3. 读：阅读器长时间阅读舒适，查阅效率高

因此界面目标不是“炫”，而是“长时间可用”。

---

## 2. Pencil 设计规则（建议直接复制到提示词）

### 2.1 信息架构规则

1. 首页只做 3 个主动作：继续听、问法、继续阅读
2. 一级导航不超过 5 个入口
3. AI 与阅读器必须可双向跳转（回答 -> 原文，原文 -> 问法）
4. 播放器在全站有统一入口，不允许每页都做一套播放器 UI

### 2.2 交互规则

1. 触控目标最小 44px
2. 主流程按钮在首屏必须可见（继续听、发送、继续阅读）
3. 所有“等待中”状态必须可视化（播放缓冲、AI 生成中、文库加载中）
4. 弱网下优先让用户“先用起来”，再做非关键加载

### 2.3 视觉规则

1. 采用安静、克制、暖纸感风格，避免科技霓虹感
2. 主色仅一个强调色，禁止同屏多个高饱和强调色争抢注意力
3. 卡片层级最多 3 级（背景层、内容层、强调层）
4. 阴影要轻，边框要细，避免厚重拟物

### 2.4 AI 问法规则

1. 回答结构固定：简述 -> 原文引用 -> 出处 -> 打开原文
2. “原文引用块”视觉优先级高于“AI 自由文本”
3. 必须显示免责声明：仅供参考，以法师原始开示为准
4. 提供“继续追问”但不能遮蔽原文入口

### 2.5 阅读器规则

1. 默认行高 1.75~1.9
2. 字号切换、字体切换、目录、进度必须一跳可达
3. 支持沉浸阅读模式（隐藏干扰控件）
4. 长文导航要有“上一讲/下一讲/章节锚点”

### 2.6 音频播放器规则

1. 迷你播放器显示：标题、进度、主控制（播/停）
2. 展开播放器显示：播放列表、倍速、睡眠、循环
3. 缓冲状态与错误状态不可复用同一个视觉样式
4. 快速切歌时，视觉反馈要先于网络完成（先切 UI，再追音频）

## 3. 可参考的优秀案例（按能力拆解）

> 下面不是让你照抄视觉，而是学习它们在“任务完成效率”上的做法

### 3.1 音频体验

1. Spotify
- 强项：迷你播放器到全屏播放器的连续性
- 可借鉴：播放控制分层、播放列表可达性

2. Apple Podcasts
- 强项：长音频进度与章节化
- 可借鉴：播放速度、睡眠、连续播放的低干扰布局

3. Audible
- 强项：有声书场景下的长时收听
- 可借鉴：断点恢复、章节切换、大按钮可触达

### 3.2 AI 知识问答体验

1. Perplexity
- 强项：回答后直接给来源
- 可借鉴：来源卡片结构、引用可信感

2. Notion AI Q&A（知识库内问答场景）
- 强项：在文档上下文内提问
- 可借鉴：问答与原文之间的低摩擦跳转

3. Readwise Reader Ask（阅读中提问）
- 强项：阅读与问答结合
- 可借鉴：原文高亮与问答联动

### 3.3 阅读器体验

1. 微信读书
- 强项：移动端长阅读舒适性
- 可借鉴：字号行距、目录、夜间模式切换

2. Kindle App
- 强项：极简阅读与稳定分页
- 可借鉴：阅读器控制层次、干扰控制

3. Medium
- 强项：正文可读性
- 可借鉴：文本宽度、段距、标题层级

---

## 4. 适合你项目的组合策略

建议采用“混合最佳实践”：

1. 播放器交互：参考 Spotify + Apple Podcasts
2. AI 引用结构：参考 Perplexity（来源优先）
3. 阅读器排版：参考 微信读书 + Kindle

目标不是做成谁，而是让用户在你站里“听-问-读”无缝切换。

## 5. 给 Pencil 的一条可直接使用的提示词

请为一个移动端音频修学网站设计 3 套 UI 方案，场景包含音频播放器、AI 问法、长文阅读器。设计必须安静、克制、暖纸感，不要科技霓虹风。首页只突出继续听、问法、继续阅读三个主动作。AI 页面回答必须是引用型结构：简述、原文引用、出处、打开原文。阅读器要支持长时阅读，具备字号、行高、目录、上下篇导航。播放器要有迷你态和展开态，弱网下也要让用户先操作成功。请输出移动端页面：首页、分类页、展开播放器、AI 页、阅读器页、我的页，并附每套方案的设计关键词与优劣比较。

## 6. 评审清单（你看图时可直接打分）

每项 0-10 分：

1. 首屏是否一眼知道“下一步做什么”
2. 音频播放操作是否低摩擦
3. AI 回答是否“引用优先”
4. 阅读器是否适合连续阅读 20 分钟以上
5. 各页面视觉是否同一系统
6. 触控是否友好（大拇指可达）
7. 弱网状态下是否有清晰反馈
8. 是否避免了花哨但无效的装饰

总分低于 55 分建议直接淘汰
