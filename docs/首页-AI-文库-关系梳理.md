# 首页 · AI · 文库 — 关系梳理

> 目标：把三个独立入口页面的职责、联动链路、信息架构、技术实现一次讲清楚。
> 来源：综合 architecture-module-responsibilities.md、pencil-uiux-rules-and-cases.md、figma-ai-creative-brief.md、ai-status-report-2026-03-31.md 的内容，去除重复、补充代码现状。

---

## 一、三页定位（一句话）

| 页面 | 入口 | 核心任务 | 关键词 |
|------|------|---------|-------|
| 首页 | index.html → main.js | 分流器 —— 让用户一个动作进入「听/问/读」 | 继续听、入口分流、修学状态 |
| AI | ai.html → ai-app.js | 引用型法问工作台 —— 基于原文回答，不是聊天机器人 | RAG 引用、原文出处、免责声明 |
| 文库 | wenku.html → wenku-app.js | 讲记索引与阅读器 —— 继续阅读、检索、沉浸阅读 | 系列目录、全文搜索、阅读进度 |

---

## 二、信息架构约束（设计铁律）

1. **首页只做 3 个主动作**：继续听、问法（→ AI）、继续阅读（→ 文库）
2. **一级导航不超过 5 个**：首页 / 有声书 / 播放 / 听经台 / 我的  
   （当前 Tab Bar 是 4+1：home、youshengshu、tingjingtai、mypage + 中间播放钮）
3. **AI 与文库必须可双向跳转**：AI 回答 → 原文（文库），文库原文 → 问法（AI）
4. **播放器全站统一**：不允许每页各做一套播放器 UI

## 四、三者协作模型（用户视角）

```
                   ┌─────────────┐
                   │    首页     │
                   │  分流入口    │
                   └──┬─────┬───┘
            继续听 │         │ 问法 / 继续阅读
                   │         │
          ┌────────▼──┐  ┌──▼─────────┐
          │  播放器    │  │            │
          │ (全站统一)  │  │            │
          └───────────┘  │            │
                         │            │
                ┌────────▼──┐  ┌──────▼──────┐
                │   AI 页   ◄──►   文库页    │
                │ 引用问答   │  │  阅读/检索   │
                └───────────┘  └─────────────┘
                      │              ▲
                      │  引用预览     │
                      └──────────────┘
```

**核心闭环：AI 回答引用原文 → 用户点击来源 → 文库预览/全文 → 文库阅读中可返回 AI 继续追问**

## 六、技术架构事实

### 6.1 三页是独立入口

```
index.html  → src/js/main.js     （SPA，内有多个 tab 视图）
ai.html     → src/js/ai-app.js   （独立页面）
wenku.html  → src/js/wenku-app.js （独立页面，内有多层路由：首页/系列/阅读器）
```

- 三个页面各有独立的 HTML、JS 入口，跳转靠 `location.href`（全页刷新，非 SPA 路由）
- 共享 theme.js（主题同步）
- AI 页共享 wenku-api.js（用于文库预览）
- 首页不导入 AI 或文库模块

### 6.2 共享 API 层

- 后端统一入口：`functions/api/[[path]].js`
- AI 路由：`functions/lib/ai-routes.js`（ask、ask-stream、search 等）
- 文库路由：`functions/lib/wenku-routes.js`（系列、文稿、搜索、阅读计数）
- 前端 API 封装：`src/js/api.js`（首页用）、`src/js/ai-client.js`（AI 用）、`src/js/wenku-api.js`（文库用，AI 页也复用）

### 6.3 数据流向

```
D1 数据库 ─── series/tracks/content/metadata
    ↓
functions/lib ─── ai-routes / wenku-routes / audio-utils
    ↓
前端 API 客户端 ─── api.js / ai-client.js / wenku-api.js
    ↓
页面渲染 ─── main.js / ai-app.js / wenku-app.js
```

## 七、当前联动薄弱点（待提升）

| # | 问题 | 当前状态 | 目标状态 |
|---|------|---------|---------|
| 1 | 首页缺"继续听/问法/继续阅读"三合一明确入口 | 只有 Tab + AI 按钮，文库入口藏在"我的"里 | 首屏首位展示三主动作 |
| 2 | AI → 文库预览已实现，但文库 → AI 缺反向链路 | 文库顶部有 AI 链接，但阅读器内无"提问"入口 | 阅读器工具栏加"对此提问"按钮 |
| 3 | 页面跳转是全刷新，用户状态不连续 | location.href 硬跳 | 可接受（PWA 缓存后跳转快），但播放状态需保持 |
| 4 | 首页不感知文库阅读进度 | 无"继续阅读"卡片 | 读取本地书签，渲染继续阅读入口 |
| 5 | AI 欢迎区固定 4 个建议问题 | 硬编码 | 可考虑根据用户阅读/听经历史动态推荐 |

## 八、下一步可选方向

### 方向 A：补全三者首页联动
- 首页增加"继续听 / 问法 / 继续阅读"三合一区块
- 优先级：高（产品核心闭环缺失）

### 方向 B：补全双向跳转
- 文库阅读器增加"对此内容提问"→ AI（带上下文）
- 优先级：中（增强探索深度）

### 方向 C：统一设计系统更新
- 根据 Pencil 设计稿重做首页/AI/文库界面
- 优先级：中低（需先确认设计方向）

### 方向 D：技术优化
- AI 预览抽屉改为更流畅的过渡
- 文库搜索结果与 AI 搜索结果互通
- 优先级：低
