# 智能体技能调用指南

本项目在 Git 仓库中内置了多类技能。智能体（Cursor / Claude Code）会自动发现并加载，但需要**明确的提示词**才能正确调用。

---

## 一、技能存放位置

| 技能类型 | 位置 | 已提交 Git |
|----------|------|------------|
| **设计技能** | `.claude/skills/impeccable`、`ui-ux-pro-max`、`radix-ui-design-system` | ✅ |
| | `.agents/skills/frontend-design` | ✅ |
| **gstack** | `.claude/skills/gstack/` | ✅ |
| **设计上下文** | `.impeccable.md` | ✅ |
| **Cursor 规则** | `.cursor/rules/ui-ux-design.mdc`、`design-skills-global.mdc` | ✅ |

---

## 二、给智能体的统一指令（放在 CLAUDE.md 或项目规则中）

复制以下内容，让智能体在适当时机自动调用技能：

```
## 技能调用规则

1. **网页浏览**：使用 gstack 的 `/browse`，不要用 mcp__claude-in-chrome__* 工具。

2. **UI/UX 设计、改版、界面优化**：
   - 先读取 .impeccable.md
   - 激活 frontend-design、Impeccable、ui-ux-pro-max 技能
   - 遵循技能中的设计准则

3. **代码审查**：使用 `/review`

4. **产品/功能讨论**：使用 `/office-hours` 或 `/plan-ceo-review`

5. **设计评审**：使用 `/plan-design-review` 或 `/design-review`

6. **QA 测试**：使用 `/qa`

7. **发布 PR**：使用 `/ship`

8. **gstack 故障**：运行 `cd .claude/skills/gstack && ./setup`
```

---

## 三、按场景的提示词模板

### 设计类

| 场景 | 提示词 |
|------|--------|
| 全站重设计 | `请使用 frontend-design 和 Impeccable 技能重新设计网站 UI/UX。先读 .impeccable.md，选定美学方向，从 index.html 和 src/css/ 开始迭代。` |
| 设计评审 | `请用 Impeccable 的 /critique 对 index.html 和主要 CSS 做 UX 评审，指出排版、配色、动效、交互问题。` |
| 局部优化 | `用 frontend-design 优化 [播放器/导航栏/卡片列表]，保持品牌调性，只改这一块。` |

### 工程类（gstack）

| 场景 | 提示词 |
|------|--------|
| 产品讨论 | `运行 /office-hours，我想讨论 [具体问题]` |
| 功能评审 | `运行 /plan-ceo-review，评审 [功能描述]` |
| 代码审查 | `运行 /review` |
| 设计评审 | `运行 /plan-design-review` |
| 发布 | `运行 /ship` |
| 网页浏览 | `运行 /browse https://...` |
| QA 测试 | `运行 /qa` |

### 直接调用 slash 命令

在输入框输入 `/` 后选择，或直接输入：

```
/office-hours
/review
/browse https://example.com
/qa
/ship
```

---

## 四、新成员克隆后

1. **拉取代码**：`git clone` 或 `git pull`
2. **gstack 初始化**（首次）：`cd .claude/skills/gstack && ./setup`
3. **设计技能**：无需额外操作，自动加载
4. **参考本文档**：按场景使用对应提示词
