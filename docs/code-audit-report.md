## 1. 今日目标

今天这轮工作的核心目标只有两个：

1. 把文库阅读流程继续收敛到更简单、更自然的路径。
2. 修复“用户体感已经读完，但系统仍当作继续阅读”的状态错判。

产品方向保持一致：

- 首页不再把用户推入一个偏重的系列页流程。
- 统一走“首页 -> 书本轻封面层 -> 阅读器”的阅读路径。
- 阅读器只保留必要动作，减少分心点。
- 文案和状态判断优先贴近真实用户感受，而不是机械依赖严格数值。

## 2. 今日实际改动

### 2.1 统一书本入口层

今天确认并继续打磨了 Wenku 的统一书本入口层：

- 首页书卡点击后，打开书本轻封面层，而不是直接进入旧系列页。
- 阅读器左上目录按钮，重新打开同一个书本轻封面层。
- 旧的 `?series=` 深链，也改为先回到首页，再打开对应书本层。

这个书本层现在承担了三个职责：

- 展示书名、讲次数、已读数。
- 展示一段更像“前言/导读”的段落式介绍。
- 提供一个明确的主 CTA，再附带目录列表。

结果是阅读入口被压缩成一个统一模型，用户不需要再理解“首页、系列页、阅读器”三套不同心智。

### 2.2 介绍文案从一句话改成段落式前言

为了避免书本层像技术面板，今天继续沿用并验证了段落式介绍生成逻辑。现在介绍文案会根据书名、文稿类型、讲次数和恢复状态，生成更接近“读前引导”的文字，而不是一条冷冰冰的摘要。

这部分已经能区分至少三种状态：

- 第一次读：引导先从第 1 讲进入。
- 读到一半：引导直接续读。
- 已基本读完：引导重新开始或回目录重读。

### 2.3 修复“99.x% 仍被视为未读完”的错判

这是今天最重要的逻辑修复。

在真实浏览器验证里，发现某篇讲记本地书签进度大约为 99.88%，界面已经显示为 100%，但系统仍给出“继续阅读”，导致首页、书本层和目录标签都出现违和感。

今天在文库书签逻辑里加入了完成度归一化规则：

- 新增完成阈值：`BOOKMARK_DONE_THRESHOLD = 99.5`
- 新增辅助函数：
	- `normalizeBookmarkPercent(percent)`
	- `hasStartedReading(percent)`
	- `hasCompletedReading(percent)`
	- `isReadingInProgress(percent)`
	- `getDisplayPercent(percent)`
- `saveBookmark()` 现在会删除 0% 书签，避免继续阅读区被无效记录污染。

这个归一化规则已经同步到以下入口：

- 首页“继续阅读”卡片
- 书本轻封面层 CTA
- 书本目录 badge
- 系列阅读统计
- 最近阅读与恢复状态判断

修复后的预期行为：

- 接近读完的书，不应继续占据首页“继续阅读”。
- 书本层 CTA 应从“继续阅读”切换为“重新开始”。
- 目录里已经读完的讲次，显示“已读”；恢复点如果属于重读场景，显示“重读”。

### 2.4 精简阅读器顶栏

在用户选择“继续精简文库阅读流程”之后，今天又做了一步 UI 减法：

- 删除阅读器顶栏分享按钮
- 移除 `wireReaderShare()` 相关逻辑

阅读器顶栏现在只保留：

- 关闭
- 目录
- 标题/进度元信息
- 设置

这次减法的目标很明确：阅读器应优先服务阅读本身，而不是承载泛操作。

## 5. 建议 Claude 重点审查的内容

请把审查重点放在下面几个方面，而不是泛泛扫全仓：

### 5.1 阅读状态归一化是否一致

重点看完成度阈值 99.5 的引入后，是否所有读取、展示、恢复入口都使用了同一套语义。

尤其关注：

- 有没有某些分支仍然直接使用原始 percent。
- 有没有“展示已读完，但逻辑仍按未读完”的残留分支。
- 删除 0% 书签后，是否会影响第一次进入或恢复逻辑。

### 5.2 历史与返回行为是否稳定

重点审查 Wenku 现在的三层路径：

- 首页
- 书本轻封面层
- 阅读器

请检查：

- `popstate` 是否会出现状态错乱。
- 从阅读器返回时，是否总能落到符合预期的首页/书本层状态。
- `?series=` 和 `?doc=` 深链是否仍然自洽。

### 5.3 旧路径与新路径是否还有冲突

今天主路径已经转向“首页 -> 书本层 -> 阅读器”，但文件里仍保留了 `renderSeries()` 这类旧路径能力。请判断：

- 它现在是否仍有必要。
- 它是否会和书本轻封面层模式形成双轨并存，增加维护成本。
- 某些按钮或逻辑是否还偷偷回到旧系列页心智。

### 5.4 阅读器设置与阅读模式切换的潜在回归

今天没有重点回归分页模式，只重点验证了滚动模式。因此请额外检查：

- 切换阅读模式时，重新打开阅读器是否可能丢失状态。
- `_pager`、滚动监听、关闭清理逻辑之间有没有残留风险。
- 重新打开同一文稿时，是否可能重复绑定事件。

### 5.5 书本层是否足够轻，但没有牺牲可用性

请从产品而不是纯代码角度看：

- 当前书本层是否已经足够简洁。
- 目录 badge 的数量和表达是否还可以再减。
- 介绍文案是否自然，是否有机械模板感。
- 顶栏删掉分享后，是否确实没有伤及核心使用场景。
